# rdish プロジェクト

料理・食材辞典アプリ。`data/dishes.json` が単一データソース。

## タグ方針

### 原則

- タグは **5次元** で構成。各次元内で粒度を統一
- 1料理あたり **4〜8個**
- 次元をまたぐ複合タグ（`イタリアチーズ` 等）は使用禁止 → 各次元のタグを個別に付与

### 次元1: 料理圏

**国レベルで統一。** 地名（都市・地方）タグは禁止。

許可:
```
フランス料理 / イタリア料理 / スペイン料理 / ポルトガル料理 / ギリシャ料理 /
ドイツ料理 / オーストリア料理 / スイス料理 / ベルギー料理 /
ポーランド料理 / ロシア料理 / ウクライナ料理 / 東欧料理 / 北欧料理 / スウェーデン料理 /
トルコ料理 / 中東料理 / レバノン料理 / アラブ料理 /
モロッコ料理 / チュニジア料理 / 北アフリカ料理 / アフリカ料理 /
インド料理 / タイ料理 / ベトナム料理 / インドネシア料理 / マレーシア料理 / シンガポール料理 /
中国料理 / 四川料理 / 広東料理 /
韓国料理 /
日本料理 / 沖縄料理 /
アメリカ料理 / ハワイ料理 / メキシコ料理 / アルゼンチン料理 / ペルー料理 / 南米料理 /
アジア料理 / 東南アジア料理
```

禁止例（→ 上位に置換）:
- ナポリ / トスカーナ料理 / プーリア / サルデーニャ / ジェノバ / ヴェネト州 / 南イタリア → `イタリア料理`
- プロヴァンス / ブルゴーニュ / ブルターニュ / ロレーヌ / リヨン / ニース → `フランス料理`
- バレンシア / カタルーニャ / バスク料理 → `スペイン料理`
- 上海料理 / 北京料理 / 揚州料理 / 江蘇料理 / 浙江料理 → `中国料理`
- ウィーン → `オーストリア料理` / ハノイ → `ベトナム料理` / ユカタン料理 → `メキシコ料理`
- アジア / アジア系 → `アジア料理` / 中華料理 → `中国料理` / 東欧 → `東欧料理`

例外: 日本の地域タグ（`東北` / `北海道` / `沖縄` 等）は郷土料理の識別に使用可

### 次元2: 料理カテゴリ

```
前菜 / 主菜 / スープ / サラダ / デザート / パスタ / 麺料理 /
ご飯もの / 軽食 / 点心 / ソース / 調味料 / パン / 菓子 / 鍋料理
```

### 次元3: 主食材

```
牛肉 / 豚肉 / 鶏肉 / 羊肉 / ジビエ / 魚介 / 貝類 / 卵 /
チーズ / 乳製品 / 野菜 / 豆類 / きのこ / ハーブ / スパイス / ナッツ / 米 / 小麦
```

禁止例（→ 統合）:
- `イタリアチーズ` / `フランスチーズ` / `ブルーチーズ` / `ソフトチーズ` 等 → `チーズ`
- `イタリア野菜` / `地中海野菜` / `夏野菜` 等 → `野菜`
- `イタリアソース` / `フランスソース` / `万能ソース` 等 → `ソース`（次元2）


## データ構造

- `data/dishes.json` — 全料理エントリ
- `types/dish.ts` — 型定義
- `public/images/dishes/<id>/` — 料理画像

## 主要スクリプト

```bash
node scripts/download-images.mjs <dish-id>  # 画像ダウンロード
python3 -c "import json; json.load(open('data/dishes.json')); print('JSON valid')"  # 検証
```
